# QR-code Share Link Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render the existing share URL as a scannable QR code on the setup screen when "Share link" is clicked, so a run can be shared phone-to-phone.

**Architecture:** A new pure module `src/qr.ts` wraps the zero-dependency `uqr` library to turn a URL into an SVG QR string (with a density flag and an over-capacity fallback). `src/setupView.ts` calls it from the existing share button's click handler and shows the SVG plus a status message.

**Tech Stack:** Vite 5, vanilla TypeScript, `uqr` (QR generation), Vitest 1 + jsdom.

## Global Constraints

- New dependency: **`uqr`** only (zero-dependency, TypeScript-native, ESM — no `@types` needed). The commit adding it MUST include both `package.json` and `package-lock.json` (CI runs `npm ci`, which requires a committed lockfile).
- Public interface (exact): `buildQrSvg(text: string): { ok: true; svg: string; dense: boolean } | { ok: false; tooLong: true }` exported from `src/qr.ts`.
- QR settings (exact): error-correction level `'M'`, `border: 2`, and `QR_DENSE_VERSION = 20` (a barcode whose selected version exceeds this is `dense`).
- Exact copy strings:
  - Dense warning: `QR is dense — hold the phone steady, or use the copied link.`
  - Too-long fallback: `Too many codes for a QR — share the copied link instead.`
- The share button's visible text becomes **`Share link`**; its `.copy-link` class is kept unchanged (existing selectors/tests depend on it).
- New setup DOM: `.qr-code` (the SVG container) and `.qr-status` (the message line), both toggled via the `hidden` attribute.
- The existing clipboard write and `.url-warning` (>2000 chars) behavior must remain unchanged; `.url-warning` and `.qr-status` are independent signals.
- Feature is setup-screen only. No QR on the play screen. No URL shortening/compression.
- **`tsc --noEmit` is a required verification gate** — `npm test` (vitest) and `npm run build` (vite) do NOT type-check.
- **Commits must NOT include any Claude / AI attribution** (`Co-Authored-By: Claude` or similar).

- [ ] **Step 5: Write the implementation**

Create `src/qr.ts`:

```ts
import { encode, renderSVG } from 'uqr';

const QR_DENSE_VERSION = 20; // tunable; higher version = denser = harder to scan phone-to-phone

export type QrResult =
  | { ok: true; svg: string; dense: boolean }
  | { ok: false; tooLong: true };

/** Encode `text` as a QR SVG string. Error-correction level M balances capacity
 *  against robustness to screen glare. Returns `dense: true` when the selected
 *  version is very high (hard to scan phone-to-phone), and `{ ok: false,
 *  tooLong: true }` when the text exceeds QR capacity. Deterministic and pure. */
export function buildQrSvg(text: string): QrResult {
  try {
    const { version } = encode(text, { ecc: 'M' });       // where over-capacity surfaces
    const svg = renderSVG(text, { ecc: 'M', border: 2 }); // quiet zone = 2 modules; default white background
    return { ok: true, svg, dense: version > QR_DENSE_VERSION };
  } catch {
    return { ok: false, tooLong: true };
  }
}
```

```ts
  copyLink.addEventListener('click', () => {
    const url =
      location.origin + location.pathname + encodeShareUrl(entries.map((e) => e.raw), currentSettings());
    shareUrl.value = url;
    urlWarning.hidden = url.length <= URL_WARN_LENGTH;
    void navigator.clipboard?.writeText(url).catch(() => {});

    const qr = buildQrSvg(url);
    if (qr.ok) {
      qrCode.innerHTML = qr.svg;
      qrCode.hidden = false;
      qrStatus.textContent = qr.dense ? QR_DENSE_MSG : '';
      qrStatus.hidden = !qr.dense;
    } else {
      qrCode.innerHTML = '';
      qrCode.hidden = true;
      qrStatus.textContent = QR_TOO_LONG_MSG;
      qrStatus.hidden = false;
    }
  });
```
